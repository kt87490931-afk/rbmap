# 룸빵여지도 어드민 개선 대화기록

**날짜**: 2025년 3월 11일  
**프로젝트**: rbmap (rbbmap.com)  
**스택**: Next.js 14, Supabase, NextAuth

---

## 1. 주요 요청사항 및 해결 내역

### 1.1 지역 관리 - 지역 추가 안됨
**문제**: https://rbbmap.com/admin/regions 에서 지역 추가가 되지 않음. 어드민 전체 페이지 작동 이상.

**원인**: OTP 인증 미완료, 에러 메시지 미표시

**해결**:
- `requireAdminOrSetup` 에러 코드 구체화 (no-session, not-admin, otp-required)
- 에러 시 OTP 인증 링크 표시
- fetch에 `credentials: 'include'` 추가
- `/api/admin/auth-check` 인증 체크 API 추가

---

### 1.2 제휴업체 관리 - 제휴업체 추가 안됨
**문제**: https://rbbmap.com/admin/partners 에서 제휴업체 추가 폼이 없음 (플레이스홀더만 있음)

**해결**: 제휴업체 추가 폼 구현 (링크, 지역, 유형, 이름, 아이콘, 연락처, 별점, 위치, 태그, 설명)

---

### 1.3 "동탄최저가" 업체 홈페이지 미노출
**문제**: AI 소개글까지 작성했으나 rbbmap.com/dongtan 에 노출되지 않음

**원인**:
- 지역 페이지가 제휴업체를 **최대 2개**만 표시
- AI 소개글(venue_intros)이 partners.desc에 동기화되지 않음

**해결**:
- `src/app/[region]/page.tsx`: 제휴업체 표시 개수 2 → 6개
- `venue_intros` 저장 시 `partners.desc` 자동 동기화 (POST, PATCH)

---

### 1.4 링크 404 및 링크 형식 오류
**문제**:
- rbbmap.com/karaoke 접속 시 404
- 제휴업체 링크가 "karaoke"만 저장되어 잘못됨
- 올바른 형식: rbbmap.com/지역/종목/업소명

**해결**:
- `/karaoke`, `/category/karaoke`, `/gangnam/category/karaoke` 등 리다이렉트 페이지 추가
- 제휴업체 링크 형식: `/dongtan/karaoke/dongtan-choigga`

---

### 1.5 등록된 제휴업체 수정 불가
**문제**: 등록된 제휴업체를 수정할 수 없음

**해결**: 수정 버튼 추가, 수정 모드에서 폼에 데이터 로드 후 PATCH 저장

---

### 1.6 제휴업체 폼 단순화 - URL 뒷부분만 입력
**요청**: "지역/종목(유형선택)/제가생각하는뒷부분url" - 뒷부분만 작성 가능하게

**해결**:
- **지역**: 드롭다운 (지역관리 데이터에서 slug 사용)
- **유형**: 드롭다운 (가라오케→karaoke, 바→bar 등 미리 정의)
- **URL 뒷부분**: 사용자만 입력 (예: dongtan-choigga)
- 링크 자동 생성: `/{regionSlug}/{typeSlug}/{urlSuffix}`

## 3. 제휴업체 링크 형식

```
rbbmap.com/지역slug/종목slug/뒷부분URL
예: /dongtan/karaoke/dongtan-choigga
```

- **지역slug**: 지역관리에서 정의 (dongtan, gangnam, suwon, jeju 등)
- **종목slug**: 유형선택과 매핑 (가라오케→karaoke, 바→bar, 퍼블릭→public 등)
- **뒷부분URL**: 사용자 입력 (영문, 하이픈 권장)
