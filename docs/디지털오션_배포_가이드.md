# DigitalOcean Droplet 배포 가이드 (룸빵여지도)

> Droplet IP: **152.42.216.40** | SSH 키 인증 사용

---

## Step 1. SSH 접속

### Windows (PowerShell 또는 CMD)
```powershell
ssh root@152.42.216.40
```
비밀번호 입력 프롬프트가 나오면 root 비밀번호를 입력합니다.

### 연결 실패 시
- DigitalOcean 대시보드에서 Droplet 상태가 "Active"인지 확인
- 네트워크/방화벽에서 22번 포트 차단 여부 확인

## Step 2. 초기 설정 스크립트 실행

### 2-1. 스크립트 복사
로컬에서 스크립트를 Droplet으로 복사합니다.

```powershell
# d:\rbmap 폴더에서 실행
scp scripts/setup-droplet.sh root@152.42.216.40:/root/
```

### 2-2. Droplet에서 실행
```bash
ssh root@152.42.216.40
chmod +x /root/setup-droplet.sh
bash /root/setup-droplet.sh
```

### 설치 항목
- Node.js 20 LTS
- Git
- PM2
- Nginx (80 → 3000 리버스 프록시)
- UFW 방화벽 (22, 80, 443 허용)

## Step 3. GitHub SSH 키 생성 (자동 배포용)

Droplet에서 GitHub Actions가 SSH로 접속하려면 **배포용 SSH 키**가 필요합니다.

### 3-1. Droplet에서 키 생성
```bash
ssh-keygen -t ed25519 -C "rbmap-deploy" -f /root/.ssh/rbmap_deploy -N ""
```

### 3-2. GitHub SSH 접속용 설정 (Droplet에서 git clone 가능하게)
```bash
# GitHub용 SSH config
mkdir -p /root/.ssh
cat >> /root/.ssh/config << 'EOF'

Host github.com
  HostName github.com
  User git
  IdentityFile /root/.ssh/rbmap_deploy
  StrictHostKeyChecking no
EOF
```

### 3-3. 공개키를 GitHub에 등록 (Deploy Key)
```bash
cat /root/.ssh/rbmap_deploy.pub
```
출력된 내용을 복사한 뒤:
- GitHub 레포 → Settings → Deploy keys → Add deploy key
- Title: `rbmap-droplet-deploy`
- Key: 붙여넣기
- **Allow write access** 체크 해제 (읽기만 필요)

### 3-4. 공개키를 authorized_keys에 추가 (GitHub Actions가 SSH 접속 가능하게)
```bash
cat /root/.ssh/rbmap_deploy.pub >> /root/.ssh/authorized_keys
```

### 3-5. 공개키를 authorized_keys에 추가 (GitHub Actions → Droplet SSH 허용)
```bash
cat /root/.ssh/rbmap_deploy.pub >> /root/.ssh/authorized_keys
```

### 3-6. 개인키를 GitHub Secrets에 등록
```bash
cat /root/.ssh/rbmap_deploy
```
출력된 **전체 내용**(`-----BEGIN ... END ...-----` 포함)을 복사한 뒤:
- GitHub 레포 → Settings → Secrets and variables → Actions
- New repository secret
- Name: `DROPLET_SSH_KEY`
- Value: 붙여넣기

## Step 6. 수동 배포 테스트 (선택)

GitHub Actions 설정 전, 수동으로 배포 흐름을 확인할 수 있습니다.

```bash
# Droplet에서
cd /var/www/rbmap
git clone https://github.com/YOUR_USERNAME/rbmap.git .
npm install
npm run build
pm2 start npm --name "rbmap" -- start
pm2 save
```

## 트러블슈팅

| 문제 | 해결 |
|------|------|
| SSH 연결 거부 | root 비밀번호 확인, Droplet 재부팅 |
| Nginx 502 Bad Gateway | PM2로 앱이 3000 포트에서 실행 중인지 확인: `pm2 list` |
| 빌드 실패 | `npm run build` 로컬에서 먼저 테스트, Node 버전 확인 |
| 환경변수 누락 | `/var/www/rbmap/.env.production` 생성 후 PM2 재시작 |
