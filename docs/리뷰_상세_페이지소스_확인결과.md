# 리뷰 상세 페이지 소스 확인 결과

**URL:** https://rbbmap.com/dongtan/karaoke/dongtankaraoke/%EB%8F%99%ED%83%84-%EA%B0%80%EB%9D%BC%EC%98%A4%EC%BC%80-mmsxnvuj  
**확인 파일:** 확인.txt (페이지 소스)

---

## 1. 현재 소스에서 보이는 것

| 항목 | 현재 값 | 기대 값 (리뷰 전용) |
|------|---------|---------------------|
| `<title>` | 룸빵여지도 \| 전국 룸싸롱·가라오케... (사이트 기본) | 새벽 3시에 제가 가수가 된 줄 알았지 뭡니까 \| 룸빵여지도 |
| `<meta name="description">` | 전국 380개 이상... 6시간마다... (사이트 기본) | 리뷰 본문 요약 또는 동탄 가라오케 이용 후기 |
| `<meta name="keywords">` | **없음** | 동탄 가라오케 이용 후기, 동탄 가라오케 리뷰, 동탄 가라오케, 룸빵여지도 (또는 제휴업체 태그) |
| `og:title` / `og:description` / `og:url` | 사이트 기본 (og:url=https://rbbmap.com) | 리뷰 제목, 리뷰 설명, 해당 리뷰 URL |
| `og:type` | website | article |
| JSON-LD `itemReviewed.name` | "동탄 가라오케 가라오케" | "동탄 가라오케" (업소명만) |

**결론:**  
리뷰 상세 URL인데 **HTML head에는 루트 레이아웃의 메타만** 들어가 있고, **리뷰 페이지의 generateMetadata 결과가 반영되지 않은 상태**입니다.  
`<meta name="keywords">`도 없습니다.

---

## 2. 원인 가능성

1. **배포 버전:** 해당 환경에 리뷰 상세용 generateMetadata(＋ keywords/description 보강)가 포함된 최신 빌드가 반영되지 않았을 수 있음.
2. **Next.js 동적 라우트:** 같은 앱이라도, 동적 세그먼트에서 생성한 메타가 초기 HTML에 안 들어가고 나중에만 적용되는 식으로 동작할 수 있음 (구글봇은 보통 첫 응답만 보므로 불리).

---

## 3. 방금 수정한 내용에서 할 보강

1. **JSON-LD `itemReviewed.name`**  
   - 현재: `post.venue` + ` ` + `typeName` → "동탄 가라오케 가라오케"  
   - 보강: 업소명만 쓰도록 `itemReviewed.name`을 **`post.venue`만** 사용.

2. **리뷰 메타가 반영되도록**  
   - 코드 상으로는 리뷰 페이지에서 `title`, `description`, `keywords`, `openGraph`를 모두 내려보내고 있음.  
   - 배포가 최신인지 확인한 뒤, 그래도 소스에 반영되지 않으면 같은 앱 내 다른 동적 페이지도 같은지 확인하고, 필요 시 Next.js 문서 기준으로 동적 메타/스트리밍 동작을 점검하는 것이 좋음.

---

## 4. 적용한 보강 (코드)

- **itemReviewed.name**  
  - `${post.venue} ${typeName}` → `post.venue`만 사용하도록 수정 (중복 "가라오케" 제거).

이 파일은 확인 결과 정리용이며, 실제 수정은 리뷰 페이지 및 레이아웃 코드에 반영됨.
