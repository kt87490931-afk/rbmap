## 2. 문제점

### 2.1 어드민과 연동되지 않은 위젯/기능

| 위젯/데이터 | 출처 | 문제 |
|-------------|------|------|
| **WidgetRowA - venue_ranks** (TOP 7 업소) | site_sections JSON / fallback 하드코딩 | 실제 partners·ranking DB와 무관. 달토·아우라 등 고정 값 |
| **WidgetRowA - keywords** | site_sections | `/search?q=강남가라오케` 링크 → **/search 페이지 없음** (404) |
| **WidgetRowA - categories** | site_sections | `/category/hostbar` 등 실제 라우트와 불일치 가능. 개수(168개 등) 하드코딩 |
| **Region Preview** | site_sections | 지역별 대표 업소 하드코딩. partners DB 미사용 |
| **WidgetRowB - timeline** | site_sections | 샘플 데이터. 실제 feed·리뷰와 연동 안 됨 |
| **WidgetRowB - notices** | site_sections | 공지 전용 어드민 없음 |
| **Hero KPI** (14 지역, 380+ 업소 등) | site_sections | DB와 동기화되지 않은 수치 가능성 |

### 2.2 복잡함 및 정보 과다

- **Region Guide**: 4개 지역 탭 + 각각 긴 설명 텍스트
- **Category Guide**: 업종 카드 + 지역별 키워드 링크
- **WidgetRowA**: 가격표 + 랭킹 + 업종 + 키워드를 한 섹션에
- **WidgetRowB**: 타임라인 + 지역 지도 + 공지 + FAQ를 한 섹션에
- **StatsBar vs Hero KPI**: 같은 성격의 숫자가 두 곳에 반복

### 2.3 SEO 이슈

- **긍정**: region_guide, category_guide 등 장문 콘텐츠는 SEO에 유리
- **부정**:
  - `/search` 404 등 깨진 링크 존재
  - 메인 H1·메타와 실제 콘텐츠가 산만해 핵심 가치 불명확
  - 중복/유사 키워드가 여러 섹션에 산재

## 3. 제거 권장 항목

| 항목 | 사유 |
|------|------|
| **WidgetRowA - keywords 블록** | `/search` 페이지 없음. 링크가 404로 이어짐 |
| **WidgetRowB - timeline** | 실제 데이터와 연동 안 된 샘플. 신뢰도 저하 |
| **StatsBar** | Hero KPI와 중복. 하나만 유지 |
| **Region Preview** | RegionsSection과 중복. 하드코딩 데이터 |
| **Hero KPI 또는 StatsBar 중 하나** | 둘 다 유지 불필요. 숫자는 DB 연동 후 한 곳에서만 표시 |
| **호스트바(/category/hostbar) 링크** | 운영 계획이 없으면 제거 |

## 4. 추가·개선 권장

### 4.1 DB 연동

| 항목 | 현재 | 개선 |
|------|------|------|
| **venue_ranks** | 하드코딩 | `/ranking` 또는 partners API 기반 실제 TOP N |
| **Region Preview** | 하드코딩 | getPartnersByRegion()로 지역별 대표 2~3개 동적 표시 |
| **Hero KPI** | site_sections | getRegions().length, partners 수, reviews 수 등 DB 집계 |

### 4.2 기능 추가

| 항목 | 설명 |
|------|------|
| **검색 페이지** | `/search` 페이지 구현 후 keywords 위젯과 연동 |
| **공지 어드민** | notice 테이블/어드민 추가 후 WidgetRowB 공지와 연동 |

### 4.3 UX·SEO 개선

| 항목 | 설명 |
|------|------|
| **메인 H1·메타 정리** | 핵심 가치를 담은 명확한 문구 (예: "전국 유흥 정보 — 지역별 업소·리뷰") |
| **섹션 순서·우선순위 조정** | 지역 선택 → 업소·리뷰 → CTA 흐름 강화, 가이드는 하단 배치 |

## 5. 실행 단계별 액션 플랜

### Phase 1 (즉시 적용)

1. **WidgetRowA keywords 제거** 또는 **/search 페이지 추가**  
   - keywords 블록 제거 시: `WidgetRowA`, `site.ts` FALLBACKS 수정  
   - /search 추가 시: `src/app/search/page.tsx` 생성, 검색 API 연동
2. **StatsBar 제거**  
   - `page.tsx`에서 StatsBar 섹션 제거  
   - site_sections stats 키는 유지 (향후 활용 가능)
3. **404 링크 점검**  
   - `/category/hostbar` 등 존재하지 않는 라우트 링크 제거 또는 라우트 추가

### Phase 2 (단기)

1. **venue_ranks DB 연동**  
   - `WidgetRowA`에서 `getPartners()` 또는 ranking API 호출  
   - 상위 N개 업소를 실제 데이터로 표시
2. **Region Preview DB 연동**  
   - `RegionPreview`에서 `getPartnersByRegion()` 사용  
   - 지역별 대표 업소 2~3개 동적 표시
3. **Hero KPI 동적화**  
   - Hero 컴포넌트에 `regionsCount`, `partnersCount`, `reviewsCount` props 전달  
   - `page.tsx`에서 DB 집계 후 전달

### Phase 3 (중기)

1. **검색 페이지 구현**  
   - `/search` 페이지, 검색 API, keywords 위젯 연동
2. **공지 어드민 추가**  
   - `notices` 테이블, 어드민 CRUD, WidgetRowB 연동
3. **섹션 축소 및 재구성**  
   - Region Guide/Category Guide 통합 또는 하단 이동  
   - WidgetRowA/B 구조 단순화

## 6. 동일 실수 재발 방지

| 상황 | 대응 |
|------|------|
| **404 링크** | 새 위젯/링크 추가 시 해당 경로 라우트 존재 여부 확인 |
| **하드코딩 데이터** | 새 위젯은 DB 또는 site_sections 연동 설계, fallback만 최소화 |
| **중복 표시** | 동일 성격 정보(숫자, 지역 목록 등)는 한 섹션에서만 표시 |
| **어드민 미연동** | 사이트에 노출되는 편집 가능 데이터는 반드시 어드민 또는 site_sections와 연동 |
