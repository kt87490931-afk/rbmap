## 1. 비교 요약

| 구분 | Cursor AI | Claude AI |
|------|-----------|-----------|
| 분석 기준 | 코드베이스 구조·데이터 흐름 | 실제 사이트 접속·렌더 결과 |
| 제거 범위 | 위젯 일부(키워드, 타임라인 등) | 섹션 단위 7개 + 중복 렌더 |

| 추가 제안 | DB 연동, 검색 페이지 | canonical, JSON-LD, 기어 버튼 숨김 |

## 3. 필요 vs 불필요 정리

### 3.1 즉시 제거 (불필요)

| 항목 | 사유 |
|------|------|
| 타임라인 | 샘플 데이터, 실제 feed와 미연동 |
| 공지사항 위젯 | 어드민 없음, 하드코딩 |
| StatsBar | Hero KPI와 동일 내용 중복 |
| WidgetRowA keywords 블록 | `/search` 404 |
| FULL REVIEWS (리뷰 0건 시) | 빈 섹션 노출 방지 |

### 3.2 유지·개선 (필요)

| 항목 | 방향 |
|------|------|
| Hero + KPI | 유지, KPI는 DB 연동 |
| 지역 카드 | DB 연동, 업소 수 정합성 |
| Region Preview | 유지, DB 연동 |
| FAQ | 유지, JSON-LD 추가 |
| 제휴업체 섹션 | 유지, 3개 미만 시 숨김 |
| CTA Strip | 유지 |

### 3.3 Cursor·Claude 의견 불일치 처리

| 항목 | Cursor | Claude | 최종 |
|------|--------|--------|------|
| **Widget A 전체** | keywords만 제거, 나머지 DB 연동 | 전체 삭제 | **keywords 제거, 가격·랭킹·업종은 DB 연동 후 유지** |
| **Region Preview** | 제거(RegionsSection과 중복) | 유지+DB 연동 | **유지 + DB 연동** (Claude 방향) |
| **가격 정보** | 각 업소 상세에서 확인 유도(Claude) vs 메인 유지(Cursor) | 삭제 | **DB 연동 가능하면 유지, 불가 시 삭제** |

## 5. 실행 체크리스트 (우선순위순)

```
■ P0 (즉시)
  □ canonical 태그 추가 (layout.tsx metadata)
  □ SectionWithSettings: 비로그인 시 기어 버튼 미렌더
  □ section-with-settings / seo-section 중복 렌더 통합
  ✓ WidgetRowA keywords 블록 제거 (2026-03 적용)
  □ /category/hostbar 링크 제거 또는 라우트 확인

■ P1 (이번 주)
  ✓ WidgetRowB 타임라인 → DB 연동 (reviewPosts 최근 5건)
  ✓ WidgetRowB 공지사항 블록 제거 (2026-03 적용)
  ✓ StatsBar 섹션 제거 (page.tsx)
  □ FullReviewSection: 리뷰 0건 시 섹션 숨김

■ P2 (단기)
  ✓ venue_ranks → getPartners(50) 상위 7개 DB 연동
  ✓ Region Preview → partners 지역별 그룹핑 DB 연동
  □ Hero KPI → DB 집계 (regions, partners, reviews)
  □ 지역 카드 업소 수 DB 반영
  □ 제휴업체 섹션: 3개 미만 시 숨김

■ P3 (중기)
  □ FAQ JSON-LD Schema
  □ Organization JSON-LD
  □ 이미지 alt 전수 점검
  □ 섹션 순서 재구성
```
