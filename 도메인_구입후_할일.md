# 도메인 구입 후 할 일

rbmap 프로젝트 배포 시 필요한 작업 목록입니다. 도메인 구입 후 순서대로 진행하세요.

---

## 1. Google Cloud Console 설정

### 1-1. OAuth 동의 화면
1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 프로젝트 선택 (rbmap 또는 brmap)
3. **APIs 및 서비스** → **OAuth 동의 화면**
4. **외부** 선택 → 앱 정보 입력 (앱 이름, 이메일 등)
5. **저장 후 계속**

### 1-2. OAuth 클라이언트 ID 생성
1. **APIs 및 서비스** → **사용자 인증 정보**
2. **+ 사용자 인증 정보 만들기** → **OAuth 클라이언트 ID**
3. 애플리케이션 유형: **웹 애플리케이션**
4. **승인된 리디렉션 URI**에 추가:
   ```
   https://실제도메인.com/api/auth/callback/google
   ```
5. **만들기** → **클라이언트 ID**, **클라이언트 보안 비밀번호** 복사

## 2. 배포 환경 환경 변수 설정

Vercel, DigitalOcean 등 배포 플랫폼의 Environment Variables에 다음을 추가:

| 변수명 | 값 | 비고 |
|--------|-----|------|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://hmuefzrorckwvexmened.supabase.co` | Supabase Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `sb_publishable_...` | Supabase Publishable key |
| `SUPABASE_SERVICE_ROLE_KEY` | `sb_secret_...` | Supabase Secret key (비공개) |
| `NEXTAUTH_SECRET` | 32자 이상 랜덤 문자열 | `openssl rand -base64 32` 로 생성 |
| `NEXTAUTH_URL` | `https://실제도메인.com` | 실제 서비스 URL |
| `GOOGLE_CLIENT_ID` | `xxxxx.apps.googleusercontent.com` | Google OAuth에서 복사 |
| `GOOGLE_CLIENT_SECRET` | `GOCSPX-xxxxxx` | Google OAuth에서 복사 |

### NEXTAUTH_SECRET 생성
```bash
openssl rand -base64 32
```

## 3. Supabase members 테이블 관리자 지정

1. 배포된 사이트에서 **한 번** Google 로그인 (`/admin` 접속)
2. Supabase **Table Editor** → **members** 테이블
3. 본인 이메일 행의 `role` 값을 `admin`으로 변경

또는 **SQL Editor**에서 실행:
```sql
UPDATE members SET role = 'admin' WHERE email = '본인이메일@gmail.com';
```

---

## 4. OTP 초기 설정

1. `https://실제도메인.com/admin` 접속
2. Google 로그인
3. `/admin/setup-otp` 페이지로 이동
4. Google Authenticator 앱으로 QR코드 스캔
5. 6자리 코드 입력 → OTP 설정 완료
6. 이후 어드민 접속 시 `/admin/verify-otp`에서 OTP 인증
