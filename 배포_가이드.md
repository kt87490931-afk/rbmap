# rbmap 배포 가이드 (DigitalOcean Droplet)

## 사전 요구사항

- Node.js 18 이상
- PM2 (`npm install -g pm2`)
- Nginx (선택, 리버스 프록시용)
- Git (코드 배포용)

---

## 2. 배포 실행

### 방법 A: 배포 스크립트 사용 (권장)

```bash
# 서버(152.42.216.40)에 SSH 접속 후 rbmap 폴더에서
cd /var/www/rbmap   # 또는 clone한 경로
chmod +x deploy.sh
./deploy.sh
```

`deploy.sh`가 자동으로 `git pull` 후 빌드·재시작합니다. **코드 변경이 사이트에 안 보이면** 서버에서 위 명령을 직접 실행하세요.

### 방법 B: 수동 배포

```bash
npm ci
npm run build

# standalone 정리
cp -r .next/static .next/standalone/.next/static
# public 폴더가 있으면
cp -r public .next/standalone/public 2>/dev/null || true

# 환경변수 복사
cp .env.production .next/standalone/.env.production 2>/dev/null || cp .env.local .next/standalone/.env.local

# PM2 실행
mkdir -p logs
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
```

## 3. Nginx 설정 (80 포트 사용 시)

```bash
# 설정 복사
sudo cp nginx-rbmap.conf /etc/nginx/sites-available/rbmap
sudo ln -sf /etc/nginx/sites-available/rbmap /etc/nginx/sites-enabled/

# default 비활성화 (충돌 시)
# sudo rm /etc/nginx/sites-enabled/default

sudo nginx -t
sudo systemctl reload nginx
```

`nginx-rbmap.conf`의 `server_name`을 서버 IP 또는 도메인으로 수정하세요.

## 7. 배포가 반영 안 될 때

**증상**: GitHub push 후에도 사이트에 변경사항이 안 보임 (예: 서치바 그대로, "지역을 선택하세요" 그대로)

**원인**: GitHub Actions가 실패했거나, Droplet에 최신 코드가 없음

**해결**:
1. 서버(152.42.216.40)에 SSH 접속
2. `cd /var/www/rbmap` (또는 프로젝트 경로)
3. `./deploy.sh` 실행 (최신 코드 pull 후 빌드·재시작)

```bash
ssh root@152.42.216.40   # 또는 사용 중인 계정
cd /var/www/rbmap
./deploy.sh
```

GitHub Actions 확인: 저장소 → Actions 탭에서 최근 워크플로우가 성공했는지 확인. 실패 시 `DROPLET_IP`, `DROPLET_USER`, `DROPLET_SSH_KEY` 시크릿 설정 여부 확인.

## 8. 도메인 연결 후

도메인 구입 후 연결 시:

1. `도메인_구입후_할일.md` 참고
2. `NEXTAUTH_URL`을 `https://도메인.com`으로 변경
3. Nginx에 SSL(Let's Encrypt) 설정
4. `./deploy.sh` 재실행
