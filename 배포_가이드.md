# rbmap 배포 가이드 (DigitalOcean Droplet)

## 사전 요구사항

- Node.js 18 이상
- PM2 (`npm install -g pm2`)
- Nginx (선택, 리버스 프록시용)
- Git (코드 배포용)

---

## 1. 서버 환경 변수 설정

프로젝트 루트에 `.env.production` 또는 `.env.local` 생성:

```env
NEXT_PUBLIC_SUPABASE_URL=https://hmuefzrorckwvexmened.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=sb_publishable_xxxx
SUPABASE_SERVICE_ROLE_KEY=sb_secret_xxxx

NEXTAUTH_URL=http://152.42.216.40
# 도메인 구입 전 Google OAuth 비워두면 어드민은 "설정 대기" 페이지 표시
NEXTAUTH_SECRET=
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
```

## 2. 배포 실행

### 방법 A: 배포 스크립트 사용 (권장)

```bash
# rbmap 프로젝트 폴더에서
chmod +x deploy.sh
./deploy.sh
```

### 방법 B: 수동 배포

```bash
npm ci
npm run build

# standalone 정리
cp -r .next/static .next/standalone/.next/static
# public 폴더가 있으면
cp -r public .next/standalone/public 2>/dev/null || true

# 환경변수 복사
cp .env.production .next/standalone/.env.production 2>/dev/null || cp .env.local .next/standalone/.env.local

# PM2 실행
mkdir -p logs
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
```

---

## 3. Nginx 설정 (80 포트 사용 시)

```bash
# 설정 복사
sudo cp nginx-rbmap.conf /etc/nginx/sites-available/rbmap
sudo ln -sf /etc/nginx/sites-available/rbmap /etc/nginx/sites-enabled/

# default 비활성화 (충돌 시)
# sudo rm /etc/nginx/sites-enabled/default

sudo nginx -t
sudo systemctl reload nginx
```

`nginx-rbmap.conf`의 `server_name`을 서버 IP 또는 도메인으로 수정하세요.

---

## 4. 유용한 PM2 명령어

| 명령어 | 설명 |
|--------|------|
| `pm2 status rbmap` | 상태 확인 |
| `pm2 logs rbmap` | 로그 실시간 확인 |
| `pm2 restart rbmap` | 재시작 |
| `pm2 stop rbmap` | 중지 |

## 6. Git 기반 배포 (자동화)

```bash
# 서버에서 최초 1회
git clone <저장소URL> rbmap
cd rbmap

# .env.production 생성 후
./deploy.sh

# 이후 업데이트 시
git pull
./deploy.sh
```

---

## 7. 도메인 연결 후

도메인을 구입해 연결한 뒤:

1. `도메인_구입후_할일.md` 참고
2. `NEXTAUTH_URL`을 `https://도메인.com`으로 변경
3. Nginx에 SSL(Let's Encrypt) 설정
4. `./deploy.sh` 재실행
