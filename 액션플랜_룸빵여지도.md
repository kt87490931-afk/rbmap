## 동일 실수 시 해결책

| 문제 | 원인 | 해결책 |
|------|------|--------|
| API 키 403/401 | 키 미설정, 잘못된 키 | `.env.local` 확인, Droplet `.env.production` 확인, 키 권한 확인 |
| Supabase 연결 실패 | URL/키 오타, CORS | 브라우저 콘솔 에러 확인, Supabase URL 끝에 `/` 없음 확인 |
| 빌드 시 환경변수 undefined | `NEXT_PUBLIC_` 누락 | 클라이언트 노출 변수는 반드시 `NEXT_PUBLIC_` 접두사 |
| 404 on 동적 라우트 | slug 한글 인코딩 | encodeURIComponent/decodeURIComponent 적용, Next.js 동적 세그먼트 정책 확인 |
| Edge Function 타임아웃 | 장시간 처리 | 작업 분할, 배치 크기 축소, 또는 Vercel Serverless Function으로 이전 |
| RLS로 인한 읽기 실패 | 정책 미설정 | Table Editor에서 RLS 디스플레이 모드로 정책 검증 |
| 배포 후 500 에러 | 서버 전용 변수 누락 | `pm2 logs rbmap` 확인, `.env.production` 변수 설정 |
| 마이그레이션 충돌 | 스키마 변경 순서 | 롤백 SQL 준비, Supabase Branch(유료) 또는 수동 복구 절차 문서화 |
